# 为什么将Python应用于金融

	银行本质上是技术公司。  --Hugo Banziger

----
## 1.1 Python是什么

	Python是一种高级的多用途编程语言，广泛用于各种非技术和技术领域。

### 1.1.1 Python简史

	20世纪80年代，荷兰人Guido van Rossum就开始了Python的开发工作，1991年Python发行第一个发行版本。
	
	值得注意的是，有两个可用的主要版本(Python2.x & Python3.x)仍在开发之中，更重要的是，它们从2008年起并行使用，这有时候令Pyhton的初学者感到困惑。
	
	Python2.x和Python3.x两个版本同时开发和使用的情况已经持续了一段时间，因为两个版本之间没有100%的代码兼容性，而且不是所有流行程序库都可以使用Python3.x。
	

### 1.1.2 Python生态系统

	Python作为一个生态系统，而不仅仅是一种编程语言，其主要特征是有大量可用的库和工具。
	
	这些库和工具通常必须在需要(例如绘图库)时倒入或者作为单独的系统进程(例如，Python开发环境)启动。
	
	IPython可能是最流行的Python交互式开发环境(IDE)，虽然它出现的时候只是一个增强的shell，但是现在已经有了许多典型的IDE特性(例如，支持性能分析和调试)。
	
	IPython有3个不同的版本：Shell版本、基于QT图形用户界面(QT console)的版本和基于浏览器的版本(Notebook)。
	
### 1.1.3 Python用户谱系

	专业软件开发人员
	
	科研开发人员和领域专家
	
	临时开发人员
	
	编程入门者(Python支持几乎所有编程风格)

### 1.1.4 科学栈

	某些库的集合被称为科学栈(Scientific Stack)，其中包括如下库：
	
	NumPy;SciPy;Matplotlib;PyTables;pandas.
	
	根据特定领域或者问题，科学栈可以通过更多的库进行扩展，这些库多半在一个或者多个上述基本库的基础上构建。但是最小公分母通常是NumPy。
	
	仅从编程语言来讲，有许多其他语言在语法和简洁性上可与Python比肩(比如，Ruby)。但Python除了具备Ruby的简洁性外，还具有科学栈(可以替代Matlab或者R等领域专用语言和工具集)。
	

## 1.2 金融中的科技

	近年来，在创新和监管的刺激下，银行和其他金融机构(如对冲基金)越来越多地发展成为技术公司而不仅仅是金融中介机构。

### 1.2.1 科技开销

	银行和金融机构共同组成了每年在科技上投入最多的行业。
	
### 1.2.2 作为业务引擎的科技

	效率的副作用之一就是，金融机构往往必须在更为复杂的产品或者交易中寻求竞争优势。

### 1.2.3 作为进入门槛的科技和人才

	一方面，在其他条件不变的情况下，随着时间的推移，科技的进步会降低成本。
	
	另一方面，金融机构持续在科技上投入巨资，以扩大市场份额、保持自身地位。
	
	在今天金融市场上取得一席之地往往需要在科技和熟练人员上大规模投资。

### 1.2.4 不断提高的速度、频率、数据量

	金融行业有一个方面最受科技进步的影响:金融交易决策和执行的速度及频率。
	
	金融行业不断提高的速度、频率和数据量带来了许多挑战:数据处理、分析速度、理论基础(缺乏稳定的一致性概念和理论)。
	
	原则上，上述挑战都只能由现代科技应对。令人惊讶的是，缺乏一致性理论的问题也常常通过技术方法处理。

### 1.2.5 实时分析的兴起

	金融行业中有一个学科的重要性正在强劲增长:金融和数据分析。

## 1.3 用于金融的Python

### 1.3.1 金融和Python语法

	科学语言三重奏:英语、数学、编程语言(Python)。
	
### 1.3.2 Python的效率和生产率

	从较高层次看，使用Python的好处可以从3个维度衡量:效率、生产率和质量。
	
	Python确保高性能的方法:范型、编译和并行化
	
### 1.3.3 从原型化到生产

	Python可以为金融应用开发和算法实现提供一致性技术框架。

## 1.4 结论

	Python作为一种语言--但更多是作为生态系统--是金融业理想的技术框架。
	
	它的特性中有许多好处，比如简洁的语法、高效的开发方法和原型化及生产的易用性等。
	
	利用Python大量的可用库和工具，似乎能够应付当今金融业中分析、数据量和频率、依从性及监管所引发的大部分问题。
	
	即使在较大型的金融机构中，它也具备提供单一、强大、一致性的框架，简化端到端开发和生产工作的潜力。
